# 文書執筆ルール（UDC2025）

最終更新日: 2026-06-14

本ドキュメントは、UDC2025 の仕様書を更新する際の執筆ルールをまとめた内部向けガイドです。

## 1. 用語・表記の統一

- カテゴリ名は [docs/00_用語集.md](docs/00_用語集.md) の正式名称を使用する。
- 「切替」ではなく「切り替え」を使用する。
- 「等」ではなく「など」を基本とする（固有名称の列挙を除く）。

## 2. ファイル参照ルール

- 依存ファイル参照は `docs/requirements_server.txt` のように、docs 配下の実パスで記載する。
- API のベースURLを記載する場合は `https://wapi.bodik.jp` を使用する。

## 3. 文体ルール

- 仕様説明は常体（〜する）で統一する。
- 箇条書きは、1項目1文を基本とする。
- 未確定事項は「現行」「将来拡張」を明示して混在を避ける。

## 4. 更新時チェックリスト

- docs 配下の関連文書で表記ゆれがないことを確認する。
- 参照先ファイル・URLが有効であることを確認する。
- 最終更新日を更新する。
