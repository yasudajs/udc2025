# データベース設計（UDC2025）

最終更新日: 2026-06-14

## 1. 方針
- 現行方針として、ローカルDBおよびサーバー側キャッシュは採用しない。
- BODIK API（WAPI）から都度取得し、ローカルに永続データを保持しない。
- そのため、本プロジェクトではDBスキーマ設計・マイグレーション設計を行わない。

## 2. データ保持方針
- データ保持はブラウザの一時利用に限定する。
- API取得データのローカル永続化（DB/ファイルキャッシュ）は行わない。

## 3. ER図（現行）
- DBを採用しないため、ER図は定義しない。

## 4. 備考
- 例外的にデータ保持要件が発生した場合は、要件定義の改定と同時に本ドキュメントを新規設計する。
