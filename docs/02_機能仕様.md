# 機能仕様（UDC2025）

最終更新日: 2026-06-14

## 1. 概要
本ドキュメントはUDC2025の機能仕様を定義する。BODIK API（WAPI）から取得した防災系オープンデータ（AED/医療機関/公衆無線LAN/指定緊急避難場所/公衆トイレ）を地図上に表示し、カテゴリ切り替え、詳細確認、外部経路案内を提供する。

対応カテゴリ: 「AED（aed）」「医療機関（hospital）」「公衆無線LAN（public_wireless_lan）」「指定緊急避難場所（evacuation_space）」「公衆トイレ（public_toilet）」
初期表示カテゴリ: AED（aed）。ユーザーは任意にカテゴリを切り替え可能。

## 2. 用語
- 一般ユーザー: 地図表示、カテゴリ切り替え、現在地表示、更新、お気に入り操作を行う利用者
- セッション: ブラウザ上での利用単位。お気に入りは localStorage に保存される。

## 5. 各機能詳細

### F-01: 地図表示/現在地
- 目的: 地図を描画し、ユーザーの現在地を表示できるようにする。
- 入力: ブラウザのGeolocation API（任意）
- 出力: 地図タイル、現在地マーカー
- 前提/制約: HTTPS必須（位置情報取得のため）。
- 例外/エラー: 位置情報拒否/失敗時は、福岡SRPセンタービル（33.5902, 130.3519）を中心に表示する。
- 受け入れ条件: 主要ブラウザ（PC/スマホ）で地図が操作可能。

### F-02: データ取得（BODIK API）
- 目的: カテゴリ（AED/医療機関/公衆無線LAN/指定緊急避難場所/公衆トイレ）別にBODIK APIからデータを取得する。
- 入力: 選択カテゴリ、地図中心座標（lat/lon）、検索半径、最大件数
- 出力: 標準化した地点データ配列（lat, lng, name, address, attrs...）
- 前提/制約: 現行はフロントエンドから直接BODIK APIを呼び出す。サーバー側キャッシュは未導入。
- 前提/制約: 現行はフロントエンドから直接BODIK APIを呼び出し、ローカルDBやサーバー側キャッシュは採用しない。
- 例外/エラー: タイムアウト/失敗時は通知表示とコンソールログで検知する。
- 受け入れ条件: 代表カテゴリでデータを取得し、マーカー描画に連携可能。

### F-03: マーカー描画/詳細
- 目的: 地点をマーカーで描画し、タップ/クリックで詳細表示。
- 入力: 標準化地点データ
- 出力: 地図上のマーカー、ポップアップ
- 表示情報: 名称、住所、電話番号、営業時間、備考（remarks）
- 前提/制約: データが未登録の項目は「データ未登録」と表示。
- 例外/エラー: 無効座標はスキップしログ。
- 受け入れ条件: 主要カテゴリで正しい位置・名称が表示される。

### F-04: カテゴリ切り替え
- 目的: カテゴリボタン操作で表示カテゴリを切り替える。
- 入力: 選択カテゴリ（単一）
- 出力: 選択カテゴリのマーカー再描画
- 前提/制約: 検索文字列による絞り込み機能は現行未実装。
- 受け入れ条件: ボタン選択時にアクティブ状態が切り替わり、対応カテゴリのデータが再取得される。

### F-05: 経路案内リンク
- 目的: 外部地図アプリへの経路リンク提供。
- 入力: 緯度経度
- 出力: 外部リンクURL
- 前提/制約: 端末/OSによりURLスキーム差分あり。
- 受け入れ条件: PC/スマホで適切なアプリ/サイトが開く。

## 6. 権限/ロール
- 一般ユーザー: 全機能を利用可能
- 管理者: 現行未実装

## 7. バリデーション方針
- 外部APIレスポンスは必須フィールド（緯度・経度）を確認し、無効なデータは描画対象外とする。
- エラー時はUI通知でユーザーへ表示する。

## 8. 国際化/日本語対応
- 文字コードUTF-8、タイムゾーンJST、曜日/日付表記

## 9. 非機能影響
- レスポンス、スループット、セキュリティ（XSS、外部API依存時の可用性）

## 10. 依存関係
- Flask 2.2.5 / Python 3.7.17
- 地図ライブラリ（Leaflet想定）
- 現行はクライアントサイド中心で動作し、ローカルデータ保持を行わない。
